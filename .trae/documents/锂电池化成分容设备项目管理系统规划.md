# 锂电池化成分容设备项目管理系统 - 规划方案

## 一、系统概述

### 1.1 项目背景

* **用户公司**：自动化锂电池化成分容设备制造商

* **系统目标**：开发简易项目管理系统，实现项目全生命周期管理

* **核心用户**：技术、采购、财务、生产、交付、售后等负责人

### 1.2 系统定位

* 轻量级B/S架构项目管理系统

* 聚焦项目全生命周期管理

* 支持多角色协同操作

***

## 二、功能需求分析

### 2.1 核心功能模块

| 模块              | 功能描述           |
| --------------- | -------------- |
| **项目管理**        | 项目创建、编辑、查询、删除  |
| **看板Dashboard** | 全局项目看板、单项目详情看板 |
| **进度管理设置**      | 里程碑、任务跟踪、进度填报  |
| **角色权限**        | 多角色登录、数据隔离与共享  |
| **文档管理**        | 项目相关文档上传、查看、下载 |
| **通知提醒**        | 任务到期、状态变更提醒    |
| **数据报表**        | 项目进度汇总、统计报表    |

### 2.2 角色权限设计

| 角色        | 权限范围            |
| --------- | --------------- |
| **技术负责人** | 技术方案、BOM、工艺文件维护 |
| **采购负责人** | 采购订单、供应商、到货进度   |
| **财务负责人** | 项目预算、费用报销、成本核算  |
| **生产负责人** | 生产计划、装配进度、质检状态  |
| **交付负责人** | 发货安排、现场调试、验收流程  |
| **售后负责人** | 售后问题、运维记录、客户反馈  |
| **管理员**   | 系统配置、全局数据管理     |

### 2.3 项目生命周期状态

```
[项目接单] → [方案设计] → [采购执行] → [生产装配] → [出厂测试] → [发货交付] → [现场调试] → [验收完成] → [售后运维]
```

## 四、看板设计

### 4.1 全局项目看板（首页Dashboard）

**入口**：系统登录后的首页

**核心指标卡片**：

| 指标     | 说明             |
| ------ | -------------- |
| 在手项目总数 | 当前所有项目中数量      |
| 进行中项目  | 状态为"进行中"的项目数   |
| 即将到期项目 | 计划交付日期 ≤ 7天的项目 |
| 已验收项目  | 状态为"验收完成"的项目数  |
| 本月新增项目 | 本月新增的项目数量      |
| 本月交付项目 | 本月完成交付的项目数量    |

**项目状态分布图表**：

* 环形图/饼图：展示各生命周期阶段的项目数量分布

* 甘特图视图：展示所有项目的进度时间线

**近期任务列表**：

* 待处理任务（当前用户权限范围内的待办事项）

* 任务到期提醒

**快捷操作**：

* 新建项目按钮

* 查看我的项目

### 4.2 单项目详情看板

**入口**：点击具体项目进入

**布局结构**：

```
┌─────────────────────────────────────────────────┐
│ 项目名称 | 客户 | 项目状态    [编辑] [删除]       │

│  └─────────┘ └─────────┘ └─────────┘ └────────┘ │
├─────────────────────────────────────────────────┤
│  项目进度时间线（横向）                           │
│  [接单] → [设计] → [采购] → [装配] → [测试]      │
│              ↓           ↓                        │
│           进行中       完成                       │
├─────────────────────────────────────────────────┤
│  各模块进度详情                                  │

│  │ 技术模块   │ 采购模块   │ 生产模块   │        │
│  │ ●负责:张三 │ ●进度:80%  │ ●进度:50%  │        │
│  │ ●状态:进行 │ ●待物料:XX │ ●装配:进行 │        │
│  └────────────┴────────────┴────────────┘        │
│  ┌────────────┬────────────┬────────────┐        │
│  │ 交付模块   │ 财务模块   │ 售后模块   │        │
│  │ ●调试:待  │ ●应收:XX   │ ●状态:正常 │        │
│  │ ●验收:待  │ ●已收:XX   │ ●维保到期: │        │
│  └────────────┴────────────┴────────────┘        │
├─────────────────────────────────────────────────┤

│  • 2026-02-25 李四 更新了生产进度为50%           │
├─────────────────────────────────────────────────┤
│  相关文档                                        │
│  • 技术方案.pdf  • 采购清单.xlsx  • 测试报告.pdf │
└─────────────────────────────────────────────────┘
```

**单项目看板核心要素**：

1. **项目概览卡片**：合同金额、已收金额、项目状态
2. **进度可视化**：横向进度条展示各阶段完成情况
3. **模块状态网格**：6大模块（技术/采购/生产/交付/财务/售后）状态一目了然
4. **操作日志**：记录所有关键操作的时间线和操作人
5. **文档快捷入口**：快速查看项目相关文档

## 五、技术选型建议

### 5.1 推荐方案

| 层级      | 技术选型                      | 说明          |
| ------- | ------------------------- | ----------- |
| **前端**  | Vue 3 + Element Plus      | 轻量、易用、支持响应式 |
| **后端**  | Node.js + Express         | 快速开发、JSON友好 |
| **数据库** | SQLite (轻量版) / PostgreSQL | 单机部署用SQLite |
| **认证**  | JWT                       | 无状态Token认证  |

### 5.2 系统架构

```
┌─────────────────┐
│   Web浏览器      │
└────────┬────────┘
         │
    ┌────▼────┐
    │  Vue3   │  (前端)
    │Element+ │
    └────┬────┘
         │ HTTP API
    ┌────▼────┐
    │ Express │  (后端)
    │  JWT    │
    └────┬────┘
         │
    ┌────▼────┐
    │ SQLite  │  (数据库)
    └─────────┘
```
