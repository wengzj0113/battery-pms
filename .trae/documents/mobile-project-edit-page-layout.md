# 移动端「新建/编辑项目」页面排版优化计划

## 目标
- 解决手机端「新建项目」页面中表单项重叠、输入框过窄、滑块/日期等控件拥挤的问题。
- 让每一项内容在手机端尽量“单独成行”（单列布局），提升可读性与可操作性。
- 保持桌面端/大屏端现有多列布局体验不变或更佳。

## 现状定位（依据代码与截图）
- 页面文件为 [ProjectEdit.vue](file:///d:/traeProjectManeger/frontend/src/views/ProjectEdit.vue)。
- “基础信息/技术信息/采购信息”已使用 `:xs="24"` 等响应式列，手机端基本正常。
- “生产信息/软件版本信息/交付信息/财务信息/售后信息”大量使用 `<el-col :span="8">`，在手机端仍然三列并排，导致控件宽度不足与重叠。
- 表单当前 `label-width="140px"`，手机端会进一步压缩输入区域，放大拥挤问题。

## 改造方案（总体）
1. **统一响应式栅格**：把后半部分所有 `:span="8"` 改为 `:xs="24" :sm="12" :md="8"`（或根据字段重要性调整），保证手机端单列、平板双列、桌面三列。
2. **手机端标签上移**：在手机端将表单 `label-position` 切换为 `top`，并将 `label-width` 变为 `auto`，避免标签占用横向空间。
3. **控件宽度统一**：确保 `el-input / el-select / el-date-picker / el-input-number` 在表单项内均为 `width: 100%`（已有的保持，缺失的补齐）。
4. **滑块在手机端更友好**：保留现有 marks，但在手机端确保滑块独占一行；必要时根据视觉效果决定是否在手机端隐藏 marks 或改为更紧凑的显示方式（例如仅显示 0/50/100 的刻度点而不显示文字）。
5. **表单底部操作按钮移动端优化（可选）**：手机端将“保存/取消”按钮改为更易点击的布局（例如横向两按钮各占 50%，或纵向堆叠），不改变桌面端表现。

## 推荐方案（结论）
我推荐采用“**方案 A：响应式栅格 + 移动端 label-top**”（也就是上面第 1-3 点作为必做，第 4-5 点按实际效果微调）。

**为什么这是最优：**
- 改动集中在 ProjectEdit.vue，不需要新页面/新组件，风险低、可控。
- 直接解决你截图里的核心问题（`:span="8"` 导致手机端三列并排、控件挤压重叠）。
- label-top 能显著提升手机可操作性：标签不再占横向宽度，输入框自然变长。

**备选方案（不推荐优先）**
- 方案 B：只做 CSS 强制单列（不改栅格）。缺点是样式耦合重、Element Plus 结构变动时容易失效。
- 方案 C：拆分为多步向导/分页面表单。体验更清爽但开发量明显更大，且会改变现有流程。

## 具体实施步骤（按文件/模块拆分）
### Step 1：补齐移动端判定（仅前端）
- 在 ProjectEdit.vue 内新增 `isMobile` 的响应式判定（基于 `window.matchMedia('(max-width: 768px)')` 或 `resize` 监听）。
- 将 `<el-form>` 的 `label-position` 与 `label-width` 改为响应式：
  - 桌面：`label-position="right"`，`label-width="140px"`（保持现状）
  - 手机：`label-position="top"`，`label-width="auto"`

### Step 2：生产信息区块栅格修复
- 将生产信息中所有 `<el-col :span="8">` 调整为 `:xs="24" :sm="12" :md="8"`。
- 对“生产进度（el-slider）”项确保在手机端占满宽度；如仍拥挤，再考虑手机端禁用 marks 文字或使用更紧凑配置。

### Step 3：软件版本信息区块栅格修复
- 将软件版本信息中所有 `<el-col :span="8">` 调整为 `:xs="24" :sm="12" :md="8"`。
- 确保输入框默认宽度为 100%（如需要通过 style 或统一 CSS 处理）。

### Step 4：交付/财务/售后区块栅格修复
- 将交付信息、财务信息、售后信息中 `<el-col :span="8">` 按同样策略调整为响应式列。
- 对 textarea（如售后记录）继续保持 `:xs="24"` 的满行布局。

### Step 5：新增/调整少量移动端样式（仅在 ProjectEdit.vue scoped 样式中）
- 增加移动端媒体查询（`@media (max-width: 768px)`）：
  - 增大表单项垂直间距（避免拥挤）
  - 若需要，调整 `el-slider` marks 的字体大小或隐藏策略
  - 优化底部按钮区域（可选）

## 验证方式
1. 本地启动前端后，用浏览器移动端模拟（宽度 360~430）打开“新建项目”页面：
   - 各区块字段不重叠
   - 输入框/选择框/日期控件宽度足够、可操作
2. 验证平板宽度（768~1024）为双列或合理排布。
3. 桌面端布局不回退（仍为原有多列体验）。
4. 功能回归：保存/取消、校验规则、编辑项目加载与提交不受影响。

## 风险与回退
- 风险：Element Plus 的 label 与栅格组合在部分尺寸下可能仍有轻微拥挤（尤其带 marks 的 slider）。
- 回退策略：优先通过“手机端 label-top + 单列栅格”解决；若 slider 仍拥挤，再对 slider 的 marks 在手机端做降级显示（仅移动端生效）。
